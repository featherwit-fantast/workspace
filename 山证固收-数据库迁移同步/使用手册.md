# 数据库增量备份脚本 — 使用手册

## 一、概述

本脚本对 `yltrs_ylcms` 和 `bond_oms` 两个数据库共 **26 张表** 执行增量备份。

增量逻辑：基于每张表的时间字段（如 `OptDate`、`create_time`、`update_time` 等），只导出指定时间点之后的数据，不重复导出历史数据。

备份产物为纯 INSERT 语句的 `.sql` 文件，可直接 `mysql < file.sql` 恢复。

## 二、文件结构

```
shell/

```

## 三、首次配置（只需一次）

编辑 `incremental_backup.sh` 第 14-29 行，填入实际的数据库连接信息：

```bash
# ---- 数据库 yltrs_ylcms ----
DB1_HOST="10.6.4.195"               # 数据库地址
DB1_PORT="3306"                      # 端口
DB1_USER="ficc"                      # 用户名
DB1_PASS="Gongxifacai@2023"          # 密码

# ---- 数据库 bond_oms ----
DB2_HOST="10.6.4.195"               # 如果和上面同库，IP 一样即可
DB2_PORT="3306"
DB2_USER="ficc"
DB2_PASS="Gongxifacai@2023"
```

如果需要改输出目录：

```bash
BACKUP_DIR="/backup"                 # 默认 /backup，按需修改
```

配置完成后即可日常使用，**日期不需要写死在脚本里**。

## 四、日常使用

### 4.1 基本语法

```bash
bash incremental_backup.sh <日期> [时间]
```

| 参数 | 格式 | 必填 | 说明 |
|------|------|------|------|
| 日期 | `YYYYMMDD` | 是 | 8 位数字 |
| 时间 | `HHMMSS` | 否 | 6 位数字，不填默认 `000000` |

### 4.2 使用示例

```bash
# 例1：备份 2026-06-05 00:00:00 之后的数据
bash incremental_backup.sh 20260605

# 例2：备份 2026-06-05 09:00:00 之后的数据
bash incremental_backup.sh 20260605 090000

# 例3：备份 2026-06-05 12:31:21 之后的数据（精确到秒）
bash incremental_backup.sh 20260605 123121
```

### 4.3 占位符替换规则

脚本内部会自动将条件模板中的占位符替换为实际值：

| 占位符 | 替换结果（以输入 `20260605 123121` 为例） | 用于字段 |
|--------|-------------------------------------------|----------|
| `{上次同步时间}` | `'2026-06-05 12:31:21'` | OptDate/OptTime/create_time/update_time/opt_time/CreateDate/HappenDate（datetime 字段） |
| `{交易日}` | `'2026-06-05'` | ValueDate/EventDate（date 字段） |

## 六、输出文件

| 项目 | 说明 |
|------|------|
| 路径 | `/backup/increment-YYYYMMDD[-HHMMSS].sql` |
| 内容 | 纯 INSERT 语句，不含 DROP TABLE / CREATE TABLE |
| 库切换 | 每段数据前自动插入 `USE yltrs_ylcms;` 或 `USE bond_oms;` |
| 编码 | 跟随数据库默认编码 |

### 文件命名规则

| 输入 | 输出文件 |
|------|----------|
| `bash incremental_backup.sh 20260605` | `/backup/increment-20260605.sql` |
| `bash incremental_backup.sh 20260605 123121` | `/backup/increment-20260605-123121.sql` |

## 七、数据恢复

### 7.1 直接导入

```bash
mysql -h10.6.4.195 -P3306 -uficc -p'Gongxifacai@2023' < /backup/increment-20260605.sql
```

文件内的 `USE database;` 语句会自动切换到正确的库，无需手动指定。

### 7.2 导入前检查

```bash
# 查看文件行数
wc -l /backup/increment-20260605.sql

# 查看前 20 行，确认 USE 语句正确
head -20 /backup/increment-20260605.sql
```

### 7.3 安全导入（建议）

```bash
# 先关闭自动提交，导入后检查再提交
mysql -h10.6.4.195 -P3306 -uficc -p'Gongxifacai@2023' -e "
  START TRANSACTION;
  SOURCE /backup/increment-20260605.sql;
  -- 检查数据无误后手动 COMMIT，或 ROLLBACK 回滚
"
```

## 八、自动化调度

### 8.1 crontab 示例

```bash
# 每天凌晨 2:30 备份昨天的增量数据
30 2 * * * bash /path/to/incremental_backup.sh $(date -d 'yesterday' +\%Y\%m\%d)

# 每天早上 6:00 备份昨天下午 15:00 之后的数据
0 6 * * * bash /path/to/incremental_backup.sh $(date -d 'yesterday' +\%Y\%m\%d) 150000

# 每天 0:30 备份当天的数据（当天从零点开始）
30 0 * * * bash /path/to/incremental_backup.sh $(date +\%Y\%m\%d)
```

> **注意**：crontab 中 `%` 需要转义为 `\%`。

### 8.2 添加日志

```bash
30 2 * * * bash /path/to/incremental_backup.sh $(date -d 'yesterday' +\%Y\%m\%d) >> /backup/cron.log 2>&1
```

## 九、常见问题

### Q1: mysqldump: command not found
```bash
# 安装 mysql-client（以 CentOS 为例）
yum install -y mysql
# 或 Ubuntu
apt install -y mysql-client
```

### Q2: Access denied
检查 `DB1_USER` 和 `DB1_PASS` 是否正确，该用户是否对目标表有 SELECT 权限。

### Q3: 备份文件为空
说明指定时间点之后没有增量数据，属于正常情况。检查输出文件大小确认。

### Q4: 某张表备份失败
- 检查该表是否存在（可能已被删除或重命名）
- 检查账号是否有该表的 SELECT 权限
- 查看 mysqldump 错误输出（去掉脚本中 `2>/dev/null` 查看具体错误）

### Q5: 如何验证备份数据量
```bash
# 统计导出的 INSERT 条数
grep -c 'INSERT INTO' /backup/increment-20260605.sql
```
